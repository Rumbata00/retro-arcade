Критерии за оценяване - Програмиране

1. GitHub Repository с публичен достъп и регулярни къмити
Проектът трябва да бъде хостван в публично GitHub repository с активно участие от всички членове на екипа.

Изисквания:

  • Публично достъпно GitHub repository
  • Регулярни къмити (commits) от всички програмисти в екипа
  • Описателни commit съобщения
  • Правилно използване на branches за различни функционалности
  • README файл с описание на проекта и инструкции за инсталация
Важно: Всички членове на екипа трябва да имат видими приноси в историята на къмитите.
2. Правилна софтуерна архитектура и добри практики
Кодът трябва да следва установени стандарти и добри практики в програмирането, осигурявайки поддържимост и разширяемост.

Изисквания:

  • Модулна архитектура с ясно разделение на отговорностите
  • Използване на подходящи design patterns
  • Спазване на SOLID принципите
  • Правилна организация на файловете и директориите
  • Използване на подходящи библиотеки и frameworks
  • Конфигурационни файлове за различни среди (development, production)
3. Дизайн на база данни
Правилно проектиране на базата данни с оптимални таблици и връзки между тях.

Изисквания:

  • Нормализирана база данни (поне до 3NF)
  • Правилни връзки между таблиците (foreign keys)
  • Подходящи типове данни за всяко поле
  • Индекси за оптимизация на заявките
  • Constraints за осигуряване на интегритета на данните
  • ER диаграма или схема на базата данни
4. Качествен код
Кодът трябва да бъде четлив, добре структуриран и да следва конвенциите на използвания език.

Изисквания:

  • Подходящо именуване на променливи, функции и класове
  • Консистентно форматиране и стил
  • Коментари за сложни части от кода
  • Избягване на code duplication (DRY принцип)
  • Правилно управление на грешки
  • Използване на linting tools
5. Добро взаимодействие с потребителя (UX/UI)
Приложението трябва да предоставя интуитивен и приятен потребителски интерфейс.

Изисквания:

  • Интуитивен и лесен за използване интерфейс
  • Responsive дизайн за различни устройства
  • Бързо време за зареждане
  • Ясни съобщения за грешки и успех
  • Accessibility стандарти
  • Консистентен дизайн през цялото приложение
6. Цялостност и завършеност на проекта
Проектът трябва да бъде напълно функционален и готов за използване.

Изисквания:

  • Всички планирани функционалности са имплементирани
  • Няма критични бъгове
  • Приложението работи стабилно
  • Документация за потребителите
  • Инструкции за инсталация и настройка
  • Демонстрация на основните функционалности
7. Адекватна обработка на грешки
Приложението трябва да обработва грешки по подходящ начин без да се срива.

Изисквания:

  • Try-catch блокове за критични операции
  • Валидация на входни данни
  • Информативни съобщения за грешки
  • Логване на грешки за debugging
  • Graceful degradation при проблеми
  • Backup стратегии за критични операции
8. Качен сайт онлайн
Проектът трябва да бъде достъпен онлайн за тестване и демонстрация.

Изисквания:

  • Работещ онлайн сайт/приложение
  • Стабилен хостинг
  • HTTPS сертификат (препоръчително)
  • Правилна конфигурация на сървъра
  • Backup и recovery план
  • Мониторинг на производителността
9. Unit Tests (Бонус критерий)
Автоматизирани тестове за проверка на функционалността на кода.

Изисквания:

  • Unit tests за основните функции
  • Test coverage поне 70%
  • Integration tests за критични workflow-ове
  • Automated testing в CI/CD pipeline
  • Mocking на външни зависимости
  • Performance tests (по желание)
Този критерий носи допълнителни точки и показва високо ниво на професионализъм в разработката.

Обща информация за оценяването

Важни бележки:

  • Всички критерии са задължителни, освен Unit Tests който е бонус
  • Качеството е по-важно от количеството функционалности
  • Екипната работа и приносът на всеки член се оценяват отделно
  • Препоръчва се редовна комуникация с преподавателя по време на разработката

Препоръчани технологии и инструменти:

  • Version Control: Git, GitHub
  • Frontend: HTML5, CSS3, JavaScript, React/Vue/Angular
  • Backend: Node.js, Python, Java, C#
  • Database: MySQL, PostgreSQL, MongoDB
  • Testing: Jest, Mocha, JUnit, NUnit
  • Deployment: Heroku, Netlify, Vercel, AWS

Този документ служи като ръководство за оценяване на програмни проекти.

За допълнителни въпроси се обърнете към преподавателя.
